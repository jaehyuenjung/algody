# ✏ 회고록

## 문제
[백준 - 부분합](https://www.acmicpc.net/problem/1806)

## 후기
```
이번 문제는 두 포인터라는 알고리즘을 이용해서 풀 수 있는 문제였다.

먼저 두 포인터 알고리즘 없이 누적합으로만 해결하는 방법을 생각해봤다.
어차피 가장 작은 길이를 갖는 값을 찾는 것이 목표이니깐 이중 for문을 이용해서 길이별로 누적합을 구하면서
조건을 만족하면 for문을 나가서 답을 구하는 것을 생각했다.

되게 좋은 풀이 방법이였다고 생각하여 기대를 부푼 마음에 제출했지만, 시간 초과가 발생하여 아쉬웠다.

연속된 합을 구하는 것이기에 누적합 알고리즘 밖에 없다고 생각하여 여러 시도를 했지만,
풀 수가 없었고 다른 알고리즘이 있다고 생각하였다.
그래서 문제에 있는 알고리즘 분류를 살펴보았고 두 포인터 알고리즘을 알게되었고 개념을 이해하고 나니
어렵지 않게 문제를 풀 수 있었다.

이번 문제를 통해 두 포인터라는 새로운 알고리즘을 알 수 있었다. 
추후 다양한 문제를 풀어 두 포인터 알고리즘을 정복할 수 있도록 노력해야겠다!
```
