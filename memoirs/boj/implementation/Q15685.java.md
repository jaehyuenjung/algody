# ✏ 회고록

## 문제
[백준 - 드래곤 커브](https://www.acmicpc.net/problem/15685)

## 후기
```
이 문제를 풀 때 두 가지 방법에 대해 고민을 했었다.

첫 번째 방법은 모든 드래곤 커브의 구조를 사전에 만들어서
입력으로 들어온 점에 반영하는 방법이었다.

구조만 알면 쉽게 풀 수는 있지만 길이 10이 나 되는 커브 구조를 일일이 구해야 한다는 점에서
알고리즘 문제를 푸는 게 아니라 드래곤 커브를 그리는 시간이 될 것 같은 느낌이 들었다..

두 번째 방법은 그대로 구현하는 건데 이마저도 점 이동이라던가 고려해야 할 것이 있었다.

두 가지 방법 중 그나마 현실적으로 풀 수 있다고 생각한 것이 두 번째 방법이라고 생각되어서
수많은 수행 착오 끝에 풀이 시간이 좀 걸리기는 했지만, 해결할 수 있었다.

뭔가 어렵게 푼 느낌이 들어서 다른 사람의 풀이 방법 찾아보았다.
여러 풀이 방법 중에 규칙을 이용한 풀이 방식이 많았다.

규칙을 이용한 방식이 훨신 구현하기 쉬웠고 코드 길이와 성능도 좋게 나왔다.

이렇게 문제를 풀 수도 있구나! 하면서도 한편으로는 조금만 더 구조적으로 생각했으면 
비슷한 방식으로 해결할 수 있지 않았나 하는 아쉬움이 남았다. 

구현 알고리즘 문제는 그대로 구현만 하면 풀 수 있지만,
그 속에서 쉽고 효율적인 방법을 찾는 것이 중요한 것 같다.
다음 구현 문제에서는 좀 더 신경을 쓰면서 풀어야겠다! 
```
