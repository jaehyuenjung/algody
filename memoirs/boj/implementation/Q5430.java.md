# ✏ 회고록

## 문제
[백준 - AC](https://www.acmicpc.net/problem/5430)

## 후기
```
이번 문제는 Dequeue 자료 구조를 구현하거나 라이브러리를 이용하면 어렵지 않게 풀 수 있는 문제였다.

하지만 문제 푸는 것과 다르게 다른 곳에서 문제가 발생하였다.

모든 예제와 질문 글에 존재하는 반례들은 통과했지만,
백준 문제를 제출하면 1%에서 바로 틀렸다는 결과가 나오는 것이 이상했다.

코드를 잘못 짤 수도 있다고 생각하여 Dequeue 자료 구조를 구현하지 말고 라이브러리를 이용해서
제출해보았지만 역시 결과가 똑같이 나왔다.

뭔가 코드상에 문제는 아니고 입력 문제일 수도 있다고 생각하였고   
여러 번의 테스트한 결과 운영체제마다 파일의 개행 처리 방식이 다르다는 것을 알았다.

윈도우의 경우 기본적으로 입력 파일을 저장할 때 CRLF 단위로 개행을 입력하지만, 
백준 채점 서버는 리눅스 기반으로 만들어졌기 때문에 LF를 개행으로 인식하여 저장한다.

따라서 여기서(PC - 윈도우 기반) 입력 파일을 만들고 실행하면 행마다 CRLF가 붙고, 바이트 단위로 파일을 읽을 때
CR이라는 문자가 더 읽어져서 백준 채점 서버랑 입력이 다르게 들어왔다.

이 문제를 해결하기 위해 BufferedReader에서 제공하는 readLine 함수를 사용하지 말고
따로 문자열을 받는 함수를 만들었더니 문제를 해결할 수 있었다.

이번 문제를 통해서 각 운영체제의 개행 처리 방식을 알 수 있었다.
파일의 문자 인코딩 방식이 여러 가지 존재한다는 것은 알고 있었지만, 개행 처리도 여러 개 있다는 것을 보고 신기했다.
앞으로 이 점을 유의하면서 문제를 풀어야겠다!
```
